Predii Knowledge Graph (PKG)

A property graph connecting every entity in the repair lifecycle with typed, weighted edges — enabling pattern detection, prediction, and semantic search

Graph Infrastructure

The PKG is the persistent intelligence store for WrenchIQ.ai. Every normalized entity lands in the graph. Every agent reads from it. Every outcome writes back to it — creating a continuous improvement loop.

Neo4j AuraDB
pgvector / Weaviate
MongoDB Event Store

Graph Layers

1

Operational

Real-time data from active repair orders, appointments, DVI sessions, and customer interactions. Updated continuously.
  • Active RO status tracking
  • Live bay assignments
  • DVI session state
  • Open TSB matching at intake
2

Historical

All closed ROs, 3C data, parts used, technician performance, and customer interaction history. Training ground for patterns.
  • YMME complaint frequency
  • Technician efficiency history
  • Part comeback rates
  • Customer visit cadence

Predictive

ML-derived nodes tagged source: PREDICTED with confidence scores. Built over Layers 1 & 2.

  • PartFailurePattern
  • RevenueOpportunity

Embedding

Every node with a summary field is embedded to 1536-dim vectors in pgvector. Enables semantic search + RAG retrieval.
  • RepairJob 3C narratives
  • TSB titles + descriptions
  • Customer concerns
  • OEM maintenance schedules

Graph Analytics

🔍 Repair Pattern Detection

Identifies PTNs with abnormal comeback rates on a specific YMME — enabling proactive brand switches and quality alerts before patterns become costly.

MATCH (p:Part)-[:NORMALIZED_TO]->(ptn:PartTerm) WITH ptn, count(p) as totalInstalls, sum(CASE WHEN ro.isComeback THEN 1 ELSE 0 END) as comebacks WHERE totalInstalls > 10 RETURN ptn.name, comebacks * 1.0 / totalInstalls AS comebackRate ORDER BY comebackRate DESC

👨‍🔧 Technician Expertise Scoring

Two-axis scoring: volume (RO count on origin/category) and quality (efficiency + ELR vs shop benchmark). Powers the Tech Assignment Agent.

ExpertiseScore(tech, origin) = 0.4 × normalize(roCount) + 0.4 × normalize(avgEfficiency) + 0.2 × normalize(avgELR / shopBenchmarkELR) Range: 0.0 – 10.0

💰 Customer Lifetime Value Projection

Forward-looking CLV combines average ticket, visit frequency, and ML-predicted retention to prioritize outreach and trust investment.

CLV = avgTicketSize × expectedVisitsPerYear × predictedRetentionYears predictedRetentionYears = f( visitFrequency, lastVisitGap, trustScore, vehicleAge )

🏷 Supplier Rebate Intelligence

Real-time gap tracking to rebate thresholds. Surfaces exact dollar amount needed and projects close date at current purchase pace.

MATCH (shop)-[:HAS_REBATE]->(rbt:Rebate {status:'IN_PROGRESS'}) RETURN rbt.name, rbt.thresholdAmount - rbt.currentSpend AS gap, rbt.rebatePercent, rbt.estimatedEarning, rbt.periodEnd ORDER BY gap ASC

🏢 Multi-Tenancy & Event Sourcing

Each ShopLocation owns its subgraph. Cross-shop aggregate queries run on anonymized rollup nodes — no shop's data is ever exposed to another. Every write to the graph is an immutable event appended to the MongoDB event log, enabling full audit trail, warranty dispute replay, and ML training data generation.

SL-001 → owns → RepairOrder, Customer, Vehicle, Technician SL-AGGREGATE → anonymized rollup → PartFailurePattern, LaborBenchmark
